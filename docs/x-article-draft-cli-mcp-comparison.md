# X 記事ドラフト — note-research-cli × MCP比較

---

## スレッド構成案

---

### 1ツイート目（フック）

```
note.comで競合調査を毎日やってる人に聞いてほしい。

CLIツール「note-research-cli」をAIエージェントのtoolとして使う構成を試したら、
コンテキストの使い道が根本から変わった。

MCP版との違いも含めて整理する🧵
```

---

### 2ツイート目（何ができるか）

```
note-research-cli でできること：

・競合ノートの検索・分析
・ライク分布・人気テーマの抽出
・自分 vs 競合のギャップ分析
・レポートをJSON/Markdownで出力
・競合分析結果から下書きを直接作成

ローカルにインストールすれば
どのプロジェクトでもAIエージェントのtoolとして使える。
```

---

### 3ツイート目（具体的なコマンド例）

```
実際の操作イメージ：

# 競合記事を分析してテーマ抽出
$ note-research competitor analyze --query "生成AI" --size 40

# 自分との差分でギャップ発見
$ note-research report gap \
  --mine-query "私のトピック" \
  --competitor-query "競合トピック" \
  --format md

# そのままJSON蓄積 → 再分析
$ note-research report needs --query "LLM" --format json > result.json

調査結果が機械可読なので、次の分析に使い回せる。
```

### 4ツイート目（認証の話）

```
認証は初回だけ：

$ note-research auth login

セッションは ~/.note-research/session.json に保存。
次からは何も考えずに実行できる。

ブラウザ自動ログイン（Playwright）・手動入力・環境変数の3モード対応。
CI/cronで回す場合は環境変数モードが安定する。
```

---

### 5ツイート目（MCP版との違い・本題）

```
同じnote.com操作でも MCP版 と CLI版 は役割が違う。

MCP版：
・Claude/Cursor/n8n からtool呼び出し
・検索・投稿・画像・コメント・統計まで網羅
・対話的に分岐探索するときに強い

CLI版：
・端末 or スクリプトで直接実行
・競合調査 + 下書き作成に特化
・バッチ・CI・cronで安定運用

機能の広さ vs 目的の絞り込み。
```

---

### 6ツイート目（コンテキスト効率の話・最大の差分）

```
エージェントのtoolとして使うときの最大の違いは
「コンテキストの使い道」。

MCP版はツール一覧・引数スキーマ・呼び分け判断を
エージェントが毎回考える必要がある。

CLI版はコマンド文字列 + 出力だけ。
ツール選択のメタ判断が消えた分、
コンテキストが「分析結果の解釈」に全振りできる。

同じ調査を繰り返すほどCLIが有利になる。
```

---

### 7ツイート目（思想の話）

```
設計の思想として：

「ローカルにインストールすれば
 どのプロジェクトでもAIエージェントのtoolとして使える」

MCP版はサーバーを立ててクライアント設定が必要。
CLI版はインストールして auth login すれば終わり。

シェルが動く環境ならどこでも使える汎用性が
エージェント連携の実質的なハードルを下げる。
```

---

### 8ツイート目（使い分けの結論）

```
まとめると：

定型の競合調査（日次・バッチ・CI）→ CLI版
対話で探索しながら投稿まで → MCP版

併用するなら：
- 日次定型調査: CLI でcron
- 例外対応・試行: MCP で対話

どちらも note-core ライブラリを共通基盤に持つので
今後のアップデートは両版に効く。
```
